# Interpreting Results

Use this file when converting report outputs into user-facing coaching.

## Core framing

Treat the review as execution and behavior analysis first.
Do not overstate indicator precision.

## Healthy day

Call a day healthy when most of these are true:

- rapid re-entry is near zero
- rapid flip is near zero
- clustered trading is near zero
- no oversized loss cluster appears
- losses stay small relative to wins
- the result is positive or only slightly negative for understandable reasons

## Dangerous day

Call a day dangerous when any of these dominate:

- rapid re-entry spikes
- clustered trading spikes
- local time-window overtrading appears
- multiple same-idea retries appear in a short window
- large losses appear in one cluster
- the day earns money but only after obvious behavior drift or recovery from chaos

## Useful labels

Use labels like:

- healthy盈利日
- dangerous盈利日
- heavy-loss day
- mixed day
- local emotional rebound day
- clustered overtrading day
- clean but risk-control-failed day

## Behavioral patterns to prioritize

When describing mistakes, prefer these buckets:

1. rapid re-entry
2. rapid flip
3. clustered trading
4. oversized single losses
5. repeated attempts in extreme RSI zones
6. short-hold reactive trades

## Safe coaching style

Be direct but not preachy.
Use patterns like:

- “结果是正的，但过程不健康。”
- “这不是不会做，而是局部节奏被带坏了。”
- “这一天值得保留的是节奏，不只是盈亏数字。”
- “赚钱不代表方法健康。”
